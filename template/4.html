问题

Chrome 异常关闭时，系统会显示该错误。

解决方法

请尝试重新启动浏览器。

如果可能，“Google Chrome”会根据您的上一次浏览会话恢复网页。

您是否安装了 Internet Download Manager？

如果您安装了 Internet Download Manager，请将其升级至 IDM 6.03 测试版（仅提供英文网页），或转至选项 > 常规下，停用“高级浏览器集成”选项。

查看冲突软件。

您也可以通过在地址栏中键入 about:conflicts 来查找冲突软件。如果计算机中存在已知会导致 Google Chrome 崩溃的软件，那么该网页将突出显示此软件以及所有用于解决冲突的建议操作。在许多情况下，冲突软件是需要用防病毒软件删除的恶意软件。

请检查您是否需要新的浏览器用户个人资料。

如果此错误消息重复出现，则说明您的浏览器用户个人资料可能已损坏。首先将“Default”子文件夹从“User Data”文件夹中移出，查看是否可解决该问题。

  1. 按键盘快捷键 Win 键 +E 打开 Windows 资源管理器。
  2. 在显示的 Windows 资源管理器窗口的地址栏中输入以下内容：

    Windows

    • Windows XP：%USERPROFILE%\Local Settings\Application Data\Google\Chrome\User Data\
    • Windows Vista/Windows 7/Windows 8：%LOCALAPPDATA%\Google\Chrome\User Data\

    Mac OS X：~/Library/Application Support/Google/Chrome/Default

    Linux：~/.config/google-chrome/Default

  3. 点击确定。
  4. 在打开的窗口中，将“Default”文件夹重命名为“Backup”。
  5. 将“User Data”文件夹中的“Backup”文件夹移到上一级的“Chrome”文件夹中。

如果仍发生此错误，请尝试创建新的用户配置文件，替换当前的配置文件。

请检查受保护系统文件是否存在问题。

  1. 转至开始菜单 > 运行。
  2. 输入以下命令：SFC.EXE /SCANNOW（务必在 SFC.EXE 与 /SCANNOW 之间留一个空格）。该命令会启动 Microsoft 实用程序，扫描和修复计算机上受保护系统文件存在的问题。

尝试打开 Google Chrome，看看是否仍然显示错误讯息。

仍然有问题？请访问帮助论坛告知我们。

请允许 Google Chrome 自动向 Google 发送使用情况统计信息和崩溃报告，以便帮助我们防止崩溃情况的发生。
